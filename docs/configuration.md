# Configuration

Reverbic follows the [XDG Base Directory specification](https://specifications.freedesktop.org/basedir-spec/latest/) (and the equivalent conventions on Windows and macOS) to separate its files into three categories: configuration, data, and cache. The locations are resolved by the `directories` crate, so they match each platform's native conventions.

| Category | Linux | Windows | macOS |
| --- | --- | --- | --- |
| Config | `~/.config/reverbic` | `%APPDATA%\Reverbic\config` | `~/Library/Application Support/Reverbic/config` |
| Data | `~/.local/share/reverbic` | `%LOCALAPPDATA%\Reverbic\data` | `~/Library/Application Support/Reverbic/data` |
| Cache | `~/.cache/reverbic` | `%LOCALAPPDATA%\Reverbic\cache` | `~/Library/Caches/Reverbic` |

> **Migration:** installs created before this change stored everything under `~/.reverbic`. On the first startup after upgrading, Reverbic automatically moves each file to its new category directory; the old folder is removed once empty. No settings are lost.

## config.json

Settings are persisted to `config.json` inside the config directory (e.g. `~/.config/reverbic/config.json` on Linux) as pretty-printed JSON, written atomically (via a temporary file and rename). Settings are managed entirely from inside the app; you normally do not edit this file by hand. Unknown or missing fields fall back to defaults, so configs from older versions keep loading.

Notable settings stored in `config.json`:

- `volume`, `volume_step`, `restore_volume` â€” playback volume and how it is restored.
- `last_selected`, `last_station`, `autoplay_last` â€” last selection and resume behavior.
- `search_history` â€” recent search queries (capped at 10 entries).
- `crossfade_secs`, `youtube_crossfade_secs`, `prebuffer_secs` â€” crossfade and buffering durations.
- `overlay_mode`, `overlay_style`, `overlay_position`, `overlay_alpha` â€” floating overlay appearance and behavior.
- `screensaver_secs` plus per-element toggles (`screensaver_clock`, `screensaver_logo`, `screensaver_visualizer`, `screensaver_recent_tracks`, `screensaver_progress_bar`, `screensaver_station_details`, `screensaver_now_playing`).
- `media_keys`, `tray_icon`, `notifications`, `discord_rpc` â€” system integration toggles.
- `duck_enabled`, `duck_volume` â€” audio ducking.
- `language`, `theme` â€” UI language and theme.
- `game_integrations` â€” game-related overlay/ducking integration (including Dota 2).
- `youtube` â€” YouTube options such as `cookies_path`, `youtube_sponsorblock`, `youtube_radio_mode`.
- `spotify` â€” Spotify integration: `client_id`, `playback_mode`, `display_name`, `is_premium`, `country`, `followers`, `stop_on_quit`, `start_on_spotify`, `radio_enabled`.
- `auto_update`, `onboarding_completed` â€” updater and first-run state.

The Spotify `refresh_token` is intentionally **not** written to `config.json` (it is marked as skipped during serialization). It is stored in the OS keyring instead â€” see below.

## Persisted data locations

Each file lives in the category directory listed above. Files and directories are created on demand, so a given path exists only once the related feature has been used.

| Category | Path (relative to category dir) | Purpose |
| --- | --- | --- |
| Config | `config.json` | Application settings (see above). |
| Data | `favorites.json` | Saved favorite stations. |
| Data | `playlists.json` | Saved station playlists. |
| Data | `youtube_bookmarks.json` | Saved YouTube videos. |
| Data | `games.json` | Optional user override for game detection; copy of the bundled `assets/games.json` you can customize. |
| Data | `library/<station>.txt` | Per-station track history; one file per station, named after a sanitized station key. |
| Data | `bin/yt-dlp` (`yt-dlp.exe` on Windows) | yt-dlp binary, downloaded automatically for YouTube support. The `bin/` directory also holds the Deno runtime and yt-dlp version/check metadata. |
| Cache | `logs/reverbic.log` | Application log (`tracing` output). |
| Cache | `youtube_url_cache.json` | Resolved YouTube stream URLs (4-hour TTL). |
| Cache | `youtube/*.m4a` | Temporary downloaded YouTube audio; wiped on every startup. |
| Cache | `librespot/` | Working directory for native Spotify playback (librespot). |

## Spotify refresh token (OS keyring)

The Spotify OAuth refresh token is stored in the operating system keyring rather than on disk. Reverbic creates the keyring entry with:

- Service: `reverbic`
- Account/entry name: `spotify_refresh_token`

On startup Reverbic loads the token from the keyring into the in-memory config; when settings are saved it writes the token back to the keyring, or deletes the credential if no token is present. If the keyring backend is unavailable, the failure is logged and Reverbic continues without the persisted token.

Reading the log is the quickest way to confirm keyring behavior:

```powershell
Get-Content "$env:LOCALAPPDATA\Reverbic\cache\logs\reverbic.log" | Select-String "keyring|refresh token"
```

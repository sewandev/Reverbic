# Configuration

Reverbic stores all of its data under a single directory: `~/.reverbic`. The location is resolved from the `USERPROFILE` environment variable, falling back to `HOME`, and finally to the current directory if neither is set. On Windows this is typically `%USERPROFILE%\.reverbic`.

## config.json

Settings are persisted to `~/.reverbic/config.json` as pretty-printed JSON, written atomically (via a temporary file and rename). Settings are managed entirely from inside the app; you normally do not edit this file by hand. Unknown or missing fields fall back to defaults, so configs from older versions keep loading.

Notable settings stored in `config.json`:

- `volume`, `volume_step`, `restore_volume` — playback volume and how it is restored.
- `last_selected`, `last_station`, `autoplay_last` — last selection and resume behavior.
- `search_history` — recent search queries (capped at 10 entries).
- `crossfade_secs`, `youtube_crossfade_secs`, `prebuffer_secs` — crossfade and buffering durations.
- `overlay_mode`, `overlay_style`, `overlay_position`, `overlay_alpha` — floating overlay appearance and behavior.
- `screensaver_secs` plus per-element toggles (`screensaver_clock`, `screensaver_logo`, `screensaver_visualizer`, `screensaver_recent_tracks`, `screensaver_progress_bar`, `screensaver_station_details`, `screensaver_now_playing`).
- `media_keys`, `tray_icon`, `notifications`, `discord_rpc` — system integration toggles.
- `duck_enabled`, `duck_volume` — audio ducking.
- `language`, `theme` — UI language and theme.
- `game_integrations` — game-related overlay/ducking integration (including Dota 2).
- `youtube` — YouTube options such as `cookies_path`, `youtube_sponsorblock`, `youtube_radio_mode`.
- `spotify` — Spotify integration: `client_id`, `playback_mode`, `display_name`, `is_premium`, `country`, `followers`, `stop_on_quit`, `start_on_spotify`, `radio_enabled`.
- `auto_update`, `onboarding_completed` — updater and first-run state.

The Spotify `refresh_token` is intentionally **not** written to `config.json` (it is marked as skipped during serialization). It is stored in the OS keyring instead — see below.

## Persisted data locations

All paths below live under `~/.reverbic`. Files and directories are created on demand, so a given path exists only once the related feature has been used.

| Path | Purpose |
| --- | --- |
| `~/.reverbic/config.json` | Application settings (see above). |
| `~/.reverbic/favorites.json` | Saved favorite stations. |
| `~/.reverbic/library/<station>.txt` | Per-station track history; one file per station, named after a sanitized station key. |
| `~/.reverbic/logs/reverbic.log` | Application log (`tracing` output). |
| `~/.reverbic/games.json` | Optional user override for game detection; copy of the bundled `assets/games.json` you can customize. |
| `~/.reverbic/bin/yt-dlp` (`yt-dlp.exe` on Windows) | yt-dlp binary, downloaded automatically for YouTube support. The `bin/` directory also holds the Deno runtime and yt-dlp version/check metadata. |
| `~/.reverbic/librespot/` | Working directory for native Spotify playback (librespot). |

Other files Reverbic may create under `~/.reverbic` include `playlists.json`, `youtube_bookmarks.json`, `youtube_url_cache.json`, and a `cache/youtube` directory.

## Spotify refresh token (OS keyring)

The Spotify OAuth refresh token is stored in the operating system keyring rather than on disk. Reverbic creates the keyring entry with:

- Service: `reverbic`
- Account/entry name: `spotify_refresh_token`

On startup Reverbic loads the token from the keyring into the in-memory config; when settings are saved it writes the token back to the keyring, or deletes the credential if no token is present. If the keyring backend is unavailable, the failure is logged and Reverbic continues without the persisted token.

Reading the log is the quickest way to confirm keyring behavior:

```powershell
Get-Content "$env:USERPROFILE\.reverbic\logs\reverbic.log" | Select-String "keyring|refresh token"
```
